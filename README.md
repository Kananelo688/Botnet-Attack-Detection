# ML-Assisted Detection of IoT Botnet DDoS Attacks: A Comparative Study

## Introduction

Software Defined Networks (SDN) have significantly advanced network programmability and management flexibility by decoupling the control plane from the data plane. However, this architectural separation concentrates critical routing intelligence within a centralised controller, making it a high-value target for Distributed Denial of Service (DDoS) attacks. The proliferation of Internet of Things (IoT) devices compounds this vulnerability: computationally constrained, infrequently updated devices can be compromised and conscripted into botnets capable of launching large-scale, coordinated DDoS campaigns.

This repository presents the implementation and evaluation of six tree-based ensemble learning algorithms — **Random Forest**, **Extra Trees**, **AdaBoost**, **XGBoost**, **LightGBM**, and **CatBoost** — for the detection of IoT botnet DDoS attacks in SDN environments. Models are benchmarked under consistent experimental conditions across two distinct datasets:

1. **CICIoT-DIAD2024** — a large-scale, realistic IoT traffic benchmark from the Canadian Institute for Cybersecurity comprising 470,464 samples across 12 DDoS attack types.
2. **Custom SDN-IoT Dataset** — a custom-generated dataset produced using the [Mininet](http://mininet.org/) network emulator and the [POX](https://github.com/noxrepo/pox) SDN controller, simulating common IoT botnet DDoS attack variants (SYN flood, UDP flood, ICMP flood) in a controlled two-tier aggregation topology.

Models are evaluated on both predictive performance (Accuracy, Precision, Recall, F1-Score) and computational resource utilisation (training time, inference latency, CPU utilisation), providing a practical basis for model selection in latency-sensitive, resource-constrained IoT security deployments.

> **Note on scope:** The Mininet/POX simulation component of this project was used exclusively for **dataset generation**. Real-time detection and mitigation integrated directly into the SDN controller is left as future work and is the intended next step for this codebase.

## Installation

### Step 1 — Install pyenv and Python 3.9 (required for POX)

Ubuntu 24.04 ships with Python 3.12. POX `gar` requires Python 3.9. Install via `pyenv`:

```bash
sudo apt update
sudo apt install -y build-essential libssl-dev zlib1g-dev \
  libbz2-dev libreadline-dev libsqlite3-dev curl \
  libncursesw5-dev xz-utils tk-dev libxml2-dev \
  libxmlsec1-dev libffi-dev liblzma-dev git

curl https://pyenv.run | bash

echo 'export PYENV_ROOT="$HOME/.pyenv"' >> ~/.bashrc
echo 'command -v pyenv >/dev/null || export PATH="$PYENV_ROOT/bin:$PATH"' >> ~/.bashrc
echo 'eval "$(pyenv init -)"' >> ~/.bashrc
source ~/.bashrc

pyenv install 3.9.18
pyenv global 3.9.18
python3 --version   # should print Python 3.9.18
```

### Step 2 — Install system packages

```bash
sudo apt install -y mininet openvswitch-switch hping3 python3-pip
```

### Step 3 — Clone and configure POX

```bash
cd ~
git clone https://github.com/noxrepo/pox
cd pox
git checkout gar
python3 pox.py --version   # verify no import errors
```

### Step 4 — Copy POX controller scripts

```bash
cp scripts/pox-controller-scripts/*.py ~/pox/ext/
```

### Step 5 — Install Python dependencies for notebooks

It is recommended to use a separate virtual environment for the ML notebooks (Python 3.10+):

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install pandas numpy scikit-learn xgboost lightgbm catboost matplotlib seaborn psutil jupyter
```

## Dataset Generation

The custom dataset is generated by running the SDN simulation and collecting OpenFlow flow statistics via the POX controller. Each traffic class (benign and each DDoS variant) is captured independently over a 20-minute simulation window, with flow features extracted at a 10-second polling interval.

**The generated dataset is compatible with the CICIoT-DIAD2024 feature schema** — features are reconstructed from OpenFlow 1.0 statistics to approximate CICFlowMeter-compatible flow records.

### Step 1 — Start POX

In a dedicated terminal:

```bash
cd ~/pox
python3 pox.py log.level --DEBUG ext.threshold_detector
```

### Step 2 — Start the Mininet topology

In a second terminal:

```bash
sudo python3 scripts/topology.py
```

Wait for the Mininet CLI prompt. Normal IoT sensor traffic from hosts `h1`–`h6` starts automatically.

### Step 3 — Trigger attack traffic (from Mininet CLI)

```
mininet> h8 python3 scripts/cnc_trigger.py --mode syn --duration 1200
mininet> h1 hping3 --syn --flood -V -p 80 192.168.1.7 &
mininet> h2 hping3 --syn --flood -V -p 80 192.168.1.7 &
mininet> h3 hping3 --syn --flood -V -p 80 192.168.1.7 &
```

Replace `--mode syn` with `--mode udp` or `--mode icmp` for other attack variants.

### Step 4 — Back up logs

```bash
cp -r logs/ logs_<attack_type>_run1/
```

Logs are written to the `logs/` directory as CSV files compatible with the notebook pipeline.

## Training and Evaluation

All model training, evaluation, and result generation is contained in the `notebooks/` directory. Launch Jupyter and run the notebooks in order:

```bash
source .venv/bin/activate
jupyter notebook
```

Notebooks cover:
- Data preprocessing and feature selection
- Dataset balancing and train/test splitting (70/30)
- Training and evaluation of all six ensemble models on both datasets
- Comparative analysis of predictive performance and computational resource utilisation

## Results Summary

| Model | CICIoT-DIAD2024 Accuracy | CICIoT-DIAD2024 F1 | Custom Dataset F1 | Inference Time (CICIoT) |
|---|---|---|---|---|
| XGBoost | **96.35%** | **96.13%** | 99.69% | 0.98s |
| Random Forest | 96.01% | 96.16% | 99.61% | 0.23s |
| LightGBM | 95.95% | 95.79% | **99.76%** | 3.45s |
| ExtraTrees | 95.52% | 95.47% | 99.45% | 0.29s |
| CatBoost | 95.18% | 95.11% | 99.61% | **0.008s** |
| AdaBoost | 94.08% | 93.21% | 99.41% | 1.21s |

Key findings:
- **XGBoost** achieves the best overall accuracy and generalisation stability across both datasets.
- **CatBoost** achieves the fastest inference time (<8ms), making it the strongest candidate for latency-sensitive deployments.
- **LightGBM** achieves the highest precision (96.64%) but has a prohibitive inference latency for real-time use.
- **AdaBoost** is the clear underperformer: slowest to train (458s on CICIoT-DIAD2024), lowest F1, and largest cross-dataset performance drop.

For full analysis see [`report.pdf`](report.pdf).

## Future Work

- Extend evaluation to all 12 DDoS attack classes in CICIoT-DIAD2024 (only 4 were available in CSV format at time of writing).
- Integrate the best-performing models directly into the POX controller for **real-time detection and automated mitigation** via OpenFlow flow rules.

- Evaluate models in multi-controller SDN topologies.
- Apply systematic hyperparameter optimisation (grid search / Bayesian optimisation).
